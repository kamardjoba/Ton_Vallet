# Исправление валидации пароля

## Проблема
```
Error: Password must contain at least one letter
```

Валидация пароля была слишком строгой - требовала обязательного наличия букв и цифр.

## Исправления

### 1. Ослаблена валидация пароля
**Файл:** `src/utils/validation.ts`

**Было:**
- Требовалось минимум 8 символов
- Обязательно должна быть хотя бы одна цифра
- Обязательно должна быть хотя бы одна буква

**Стало:**
- Требуется минимум 8 символов
- Можно использовать любые символы (буквы, цифры, специальные символы)
- Пользователь сам выбирает уровень безопасности

### 2. Улучшены подсказки для пользователя
**Файлы:**
- `src/components/RestoreWallet.tsx`
- `src/components/InitializeWallet.tsx`

Добавлены подсказки:
- Placeholder: "Password (minimum 8 characters, any characters allowed)"
- Подсказка под полем: "Use at least 8 characters. Letters, numbers, and symbols are allowed."

## Требования к паролю

### Минимальные требования:
- **Минимум 8 символов**
- Можно использовать любые символы (буквы, цифры, специальные символы)

### Рекомендации для безопасности:
- Используйте длинные пароли (12+ символов)
- Комбинируйте буквы (верхний и нижний регистр), цифры и символы
- Избегайте простых паролей (12345678, password и т.д.)

## Ошибка "Unexpected token 'export'"

Эта ошибка в `webpage_content_reporter.js` не связана с нашим кодом. Это скорее всего:
- Расширение браузера
- Скрипт Telegram WebApp
- Другой внешний скрипт

**Можно игнорировать** - это не влияет на работу приложения.

## Результат

Теперь пользователи могут:
- Использовать пароли только из цифр (если хотят)
- Использовать пароли только из букв
- Использовать любую комбинацию символов
- Главное требование - минимум 8 символов

Восстановление кошелька должно работать корректно с любым паролем длиной от 8 символов.
